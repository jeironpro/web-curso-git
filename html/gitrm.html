Git rm

dictionaryDefinición

El comando git rm se utiliza para eliminar archivos del área de trabajo y del área de preparación (staging area). Cuando eliminas un archivo con git rm, Git registra su eliminación para que quede reflejada en el próximo commit. Es una manera segura y controlada de eliminar archivos del proyecto manteniendo la trazabilidad en el historial del repositorio.

keyboard_command_keyComandos

  • git rm <archivo>arrow_right

    Elimina el archivo del repositorio y del directorio de trabajo.

  • git rm --cached <archivo>arrow_right

    Elimina el archivo solo del área de preparación, pero lo mantiene en el directorio local.

  • git rm -r <directorio>arrow_right

    Elimina recursivamente un directorio completo y sus archivos rastreados.

  • git rm --f <archivo>arrow_right

    Forza la eliminación del archivo incluso si tiene cambios sin confirmar.

terminal Ejecutar

$ echo "Git no es GitHub" > git.txt
$ git status
$ git add git.txt
$ git status
$ git rm -f git.txt
$ git status
