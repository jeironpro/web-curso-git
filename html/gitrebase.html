Git rebase

dictionaryDefinición

El comando git rebase se utiliza para aplicar los cambios de una rama sobre otra, reescribiendo el historial de commits. Permite mantener un historial lineal y limpio, evitando merges innecesarios y facilitando la revisión de cambios. Es especialmente útil antes de hacer un push de ramas de características a la rama principal.

keyboard_command_keyComandos

  • git rebase mainarrow_right

    Aplica los commits de la rama actual sobre la rama main.

  • git rebase -i HEAD~3arrow_right

    Rebase interactivo de los últimos 3 commits para editar, combinar o reordenar.

  • git rebase --continuearrow_right

    Continúa un rebase que se ha detenido por conflictos después de resolverlos.

  • git rebase --abortarrow_right

    Aborta el rebase y regresa al estado original antes de iniciarlo.

terminal Ejecutar

$ git branch
$ git checkout backend
$ echo "GitHub no depende de Git" > github.txt
$ git status
$ git add github.txt
$ git commit -m "Nuevo archivo de texto"
$ git checkout main
$ echo "print('Git es impresionante')" >> script.py
$ git status
$ git add script.py
$ git commit -m "Nuevo print en script.py"
$ git checkout backend
$ git rebase main
$ git log --oneline
$ git push
$ git push --set-upstream origin backend
