Git revert

dictionaryDefinición

El comando git revert se utiliza para crear un nuevo commit que deshace los cambios introducidos por un commit anterior. A diferencia de git reset, git revert no altera el historial de la rama, por lo que es seguro para usar en ramas compartidas. Es útil para corregir errores o revertir cambios sin reescribir el historial.

keyboard_command_keyComandos

  • git revert HEADarrow_right

    Revierte el último commit realizado en la rama actual.

  • git revert -n <hash>arrow_right

    Prepara la reversión en el área de staging sin crear inmediatamente un commit.

  • git revert --no-edit <hash>arrow_right

    Revierte un commit usando el mensaje por defecto sin abrir el editor.

terminal Ejecutar

$ git log --oneline
$ git revert b572aa5
