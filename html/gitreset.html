Git reset

dictionaryDefinición

El comando git reset se utiliza para deshacer cambios en el área de preparación (staging) o en los commits locales. Permite mover HEAD a un commit anterior y opcionalmente modificar el índice y el directorio de trabajo según el modo usado. Es útil para corregir errores recientes o reorganizar commits antes de hacer push.

keyboard_command_keyComandos

  • git reset --soft <hash>arrow_right

    Mueve HEAD a un commit anterior, manteniendo los cambios en staging.

  • git reset --mixed <hash>arrow_right

    Mueve HEAD a un commit anterior y quita los cambios del área de staging, pero los mantiene en el directorio de trabajo (por defecto).

  • git reset --hard <hash>arrow_right

    Mueve HEAD a un commit anterior y descarta todos los cambios en staging y directorio de trabajo. Usar con precaución.

  • git reset HEAD <archivo>arrow_right

    Quita un archivo específico del área de staging, manteniendo los cambios en el directorio de trabajo.

terminal Ejecutar

$ echo "import os" > script.py
$ git status
$ git add script.py
$ git status
$ git log --oneline
$ git reset --soft b572aa5
$ git status
$ git reset --hard b572aa5
$ git status
