Git restore

dictionaryDefinición

El comando git restore se utiliza para descartar cambios en archivos del área de trabajo o del área de preparación. Permite volver al último estado guardado en Git, ya sea en el último commit o en la versión que está actualmente registrada en el área de preparación. Es una herramienta útil para revertir errores antes de hacer un commit, sin afectar el historial del repositorio.

keyboard_command_keyComandos

  • git restore <archivo>arrow_right

    Descarta los cambios locales del archivo que aún no has hecho commit.

  • git restore --staged <archivo>arrow_right

    Saca un archivo del área de preparación, sin perder los cambios en el directorio de trabajo.

  • git restore --source=<commit> <archivo>arrow_right

    Restaura el archivo desde un commit específico.

  • git restore .arrow_right

    Descarta todos los cambios no confirmados en el proyecto actual.

terminal Ejecutar

$ echo "console.log('Git es impresionante');" > main.js
$ git status
$ git restore app.js
$ git status
