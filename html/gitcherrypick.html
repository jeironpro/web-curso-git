Git cherry-pick

dictionaryDefinición

El comando git cherry-pick se utiliza para aplicar un commit específico de otra rama sobre la rama actual. Permite seleccionar cambios puntuales sin fusionar ramas completas, facilitando la incorporación de correcciones o mejoras aisladas. Es útil para aplicar hotfixes o cambios específicos sin afectar el historial general.

keyboard_command_keyComandos

  • git cherry-pick <hash>arrow_right

    Aplica el commit especificado por su hash sobre la rama actual.

  • git cherry-pick <hash1> <hash2>arrow_right

    Aplica múltiples commits consecutivos en orden sobre la rama actual.

  • git cherry-pick -x <hash>arrow_right

    Aplica un commit e incluye en el mensaje de commit la referencia al commit original.

  • git cherry-pick --abortarrow_right

    Detiene un cherry-pick en conflicto y revierte los cambios aplicados hasta el momento.

terminal Ejecutar

$ git branch
$ git log --oneline
$ git checkout backend
$ git status
$ git cherry-pick b572aa5
