Git bisect

dictionaryDefinición

El comando git bisect se utiliza para encontrar el commit que introdujo un error mediante búsqueda binaria. Permite identificar rápidamente el commit problemático dentro de un historial largo probando versiones “buenas” y “malas”. Es útil para depuración eficiente en proyectos grandes.

keyboard_command_keyComandos

  • git bisect startarrow_right

    Inicia el proceso de búsqueda binaria para localizar un commit problemático.

  • git bisect badarrow_right

    Marca el commit actual como “malo”, es decir, que contiene el error.

  • git bisect good <hash>arrow_right

    Marca un commit conocido como “bueno”, que no contiene el error.

  • git bisect resetarrow_right

    Finaliza el proceso de búsqueda y regresa a la rama original.

terminal Ejecutar

$ echo "print('Hello world')" > script.py
$ git status
$ git add script.py
$ git commit -m "Commit bueno"
$ echo "print(10/0)" > script.py
$ git status
$ git add script.py
$ git commit -m "Commit malo. División por cero"
$ echo "print('Python es un buen lenguaje')" > script.py
$ git status
$ git add script.py
$ git commit -m "Otro commit bueno"
$ git log --oneline
$ git bisect start
$ git bisect bad 67cd88d
$ git bisect good a417ad9
$ git bisect bad
$ git bisect run python script.py
$ git bisect reset
