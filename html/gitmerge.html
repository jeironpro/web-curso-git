Git merge

dictionaryDefinición

El comando git merge se utiliza para combinar los cambios de una rama en la rama actual. Permite integrar desarrollos paralelos, manteniendo el historial de commits de ambas ramas. Dependiendo de los cambios, puede generar un merge automático o requerir resolver conflictos manualmente.

keyboard_command_keyComandos

  • git merge <rama>arrow_right

    Combina la rama especificada con la rama actual.

  • git merge --no-ff <rama>arrow_right

    Realiza un merge creando siempre un commit de merge, incluso si se podría hacer fast-forward.

  • git merge --abortarrow_right

    Detiene un merge en conflicto y regresa al estado anterior al merge.

  • git merge --squash <rama>arrow_right

    Combina los cambios de la rama especificada en un solo commit, sin crear un commit de merge.

terminal Ejecutar

$ git branch
$ echo "console.log('Fusion de ramas')" >> main.js
$ git status
$ git add main.js
$ git status
$ git commit -m "Cambios en main.js desde la rama backend"
$ git checkout main
$ git branch
$ git merge backend
$ git push
