Prueba final

  1. git init - Crear un nuevo repositorio local
    • Crea una carpeta llamada proyecto_git.
    • Entra a ella y ejecuta git init.
    • Verifica que se haya creado la carpeta oculta .git.
  2. git clone - Clonar un repositorio existente
    • Clona el repositorio oficial (recuerda haber creado la conexión SSH entre Git y GitHub).
      • git clone git@github.com:usuario/repo.git
  3. git config - Configurar la identidad del usuario
    • Configura tu nombre y correo globalmente:
      • git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
    • Comprueba la configuración con git config --list.
  4. git remote - Conectar el repositorio local con uno remoto
    • Crea un repositorio vacío en GitHub llamado proyecto_git.
    • Enlázalo con tu repositorio local:
      • git remote add origin git@github.com:usuario/proyecto_git.git
    • Verifica la conexión con git remote -v.
  5. git add - Preparar archivos para el commit
    • Crea un archivo llamado README.md.
    • Añádelo al área de preparación:
      • git add README.md
    • Verifica el estado con git status.
  6. git restore - Revertir cambios no deseados
    • Modifica un archivo.
    • Ejecuta git restore nombre_del_archivo para revertir los cambios.
  7. git rm - Eliminar archivos del repositorio
    • Crea un archivo nota.txt y luego elimínalo con:
      • git rm nota.txt
    • Confirma la eliminación con un commit.
  8. git mv - Mover o renombrar archivos
    • Renombra README.md a INSTRUCCIONES.md con:
      • git mv README.md INSTRUCCIONES.md
  9. git clean - Eliminar archivos no rastreados
    • Crea un archivo temporal prueba.tmp.
    • Elimínalo con:
      • git clean -f
  10. git commit - Guardar los cambios en el historial
    • Realiza un commit con un mensaje descriptivo:
      • git commit -m "Primer commit"
  11. git log - Consultar el historial de commits
    • Ejecuta git log y observa los autores, fechas y mensajes.
  12. git show - Ver detalles de un commit específico
    • Muestra la información del último commit con:
      • git show
  13. git diff - Ver las diferencias entre versiones
    • Modifica un archivo y ejecuta:
      • git diff
    • Observa las líneas añadidas o eliminadas antes de hacer commit.
  14. git blame - Ver quién modificó cada línea de un archivo
    • Usa git blame INSTRUCCIONES.md para ver el autor de cada línea.
  15. git reflog - Ver todas las acciones recientes del repositorio
    • Ejecuta git reflog para ver los movimientos de HEAD, incluso commits borrados.
  16. git reset - Volver a un estado anterior
    • Haz un commit de prueba.
    • Luego deshazlo con:
      • git reset --soft HEAD~1
  1. git revert - Deshacer un commit creando uno nuevo
    • Revierte el último commit con:
      • git revert HEAD
  2. git bisect - Buscar el commit que introdujo un error
    • Inicia una búsqueda con:
      • git bisect start
git bisect bad
git bisect good
  3. git stash - Guardar temporalmente los cambios sin hacer commit
    • Guarda tus cambios con:
      • git stash
    • Recupéralos después con git stash pop.
  4. git branch - Crear o listar ramas
    • Crea una nueva rama llamada experimento con:
      • git branch experimento
  5. git checkout - Cambiar de rama
    • Cámbiate a la rama experimento con:
      • git checkout experimento
  6. git switch - Cambiar de rama (alternativa moderna a checkout)
    • Vuelve a la rama principal con:
      • git switch main
  7. git merge - Fusionar ramas
    • Une los cambios de experimento en main con:
      • git merge experimento
  8. git rebase - Reaplicar commits sobre otra rama
    • Ejecuta git rebase main desde otra rama para reorganizar su historia.
  9. git cherry-pick - Copiar un commit específico a otra rama
    • Selecciona un commit del historial y aplícalo con:
      • git cherry-pick <hash>
  10. git fetch - Descargar información del remoto sin combinarla
    • Ejecuta git fetch origin para traer referencias nuevas sin mezclarlas.
  11. git pull - Descargar y fusionar cambios remotos
    • Actualiza tu rama local con:
      • git pull origin main
  12. git push - Subir commits al repositorio remoto
    • Envía tus cambios al servidor con:
      • git push origin main
  13. git tag - Marcar versiones específicas del proyecto
    • Crea una etiqueta para una versión estable:
      • git tag v1.0
git push origin v1.0
  14. git submodule - Incluir otro repositorio dentro del tuyo
    • Añade un submódulo con:
      • git submodule add https://github.com/git/git submodulo_git
  15. git archive - Crear un paquete del proyecto
    • Genera un archivo comprimido del proyecto:
      • git archive -o proyecto.zip HEAD
  16. git gc - Limpiar y optimizar el repositorio
    • Ejecuta git gc para eliminar datos innecesarios y optimizar el repositorio.
